Messaging POC:

  • send a message using the below form (this form does what the mobile app would do, it sends a message to the "sendmessage" microservice with the user ID of the current user, the GP's email address, and the message to be sent). For testing, use your own email address as the GP email address
  • when submitted, you should receive an email at the specified address, coming from [userid]@sendgrid.mariomenti.com
  • reply to that email
  • the reply will be sent to the "getmessage" endpoint, where it's being processed (e.g. remove quoted replies etc.), and then saved to Redis
  • the received reply can be retrieved from Redis via the URL /listredis?userid=[userid]", where userid is whatever you entered in the form below (in reality, this will be retrieved by the mobile app and displayed there). As an example, http://35.189.97.237/listredis?userid=mariotest1 - but to see your GP reply, replace mariotest1 with your user id. Only the most recent reply for each user ID is currently stored in Redis (as it's a POC, for the real thing we almost certainly won't want to use Redis). This may take a few minutes depending on how long it takes for the email to arrive, if you see a message "userid not found" just wait a bit and try again)

BTW the source for all this mess is at https://github.com/mmenti/messaging-poc :) - It's all running under Kubernetes on the Google Cloud Platform/ Google Container Engine.
